Resume Search Engine

The resume search engine is a platform that allows for ingestion and searchability of resumes by content, name, and location. It is currently being utilized by a recuriting firm in NYC.

The technology behind the system is Apache HTTP, PHP, MySQL, and Apache Solr.

Apache HTTP and PHP allow for a hosted front-end service that allows for searchability, the ability to upload new resumes into the system, administrative management of users and permissions, adding metadata to resumes, and downloading resumes. MySQL is used to store metadata for the documents and the full text of the content of the resume. Initially InnoDB FULLTEXT Index was implemented due to the initial set of documents being the range of 50,000; however, the searchability features were lacking on resume content. Searchability is achieved through indexing with Apache Solr to allow for rich full-text search features on resume content.

Resumes are added on a regular basis and due to searches being conducted through Apache Solr, which requires the index to be created from the contents of the MySQL database, a scheduled reindexing is done to manage deltas every hour.

Parking Garage Content Management System

The Parking Garage System is a system that allows the management and visualization of parking data in parking garages. It is currently utilized by a chain of parking garages in NYC.

The technology behind the system is Java, Servlets, Java Servlet Pages (JSP), and MySQL.

The inherenet system in the garages that runs the application for interactions with customers and payment management is written in .NET. The content management system is managed utilizing Java. Data is synced from the databases in the garages to the central database as a scheduled task. The data for car check-ins, check-outs, and other associated metadata is logically synced based on calculated deltas to reduce network overhead. Also, the rates in the garages is managed through the central database and the changes are propogated to the garages via the syncing process.

The data is visualized through an Apache Tomcat web service utilizing Java Servlets and JSP. Garage rate management is also provided through this service.
